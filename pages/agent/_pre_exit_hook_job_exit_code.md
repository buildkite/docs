> 🚧 Hooks that run after `command` can change the job exit code
> If a `post-command`, `pre-artifact`, `post-artifact`, or `pre-exit` hook can fail, its exit code may replace the command's exit code as the final job result. This can affect automatic [retry](/docs/pipelines/configure/retry) rules that match on specific exit codes. To avoid this, ensure these hooks exit with code `0`, or handle errors within the hook itself.
