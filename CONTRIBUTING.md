# Contributing to the Buildkite Docs

Thanks for deciding to contribute to the [Buildkite Docs](https://buildkite.com/docs/). This guide covers what you need to know to make a contribution.

## Getting started

1. Fork and clone the repository (see the [README](./README.md) for setup instructions).
1. Create a new branch for your changes.
1. Make your changes, following the style guides below.
1. Preview your changes locally by running the development server.
1. Open a pull request against `main`.

## Style guides

Before writing or editing content, review the relevant style guides in the [`/styleguides`](./styleguides) folder:

- **[Writing style](./styleguides/writing-style.md):** Language, tone, voice, terminology, and formatting standards for all Buildkite documentation.
- **[Markdown syntax style](./styleguides/markdown-syntax-style.md):** The specific Markdown syntax used to write the docs, including headings, lists, links, callouts, tables, code blocks, and screenshots.
- **[Working with the docs site](./styleguides/working-with-docs-site.md):** How to add new pages, configure navigation, work with the file structure, and understand the site build process.
- **[YAML documentation](./styleguides/yaml.md):** Guidelines for writing about and presenting YAML in documentation, including correct terminology and code formatting conventions.

If something isn't covered in these guides, refer to the [Google developer documentation style guide](https://developers.google.com/style), followed by the [Microsoft Style Guide](https://docs.microsoft.com/en-us/style-guide/welcome/).

## Quick tips

- Use US English and active voice.
- Use sentence case for headings.
- Use serial commas.
- Run the site locally and check your changes render correctly before opening a PR.
